# 重新定义 Todo：基于生活场景的四类任务

> 传统 Todo 列表把所有事情堆在一起，完成一个划掉一个。但生活中的事务本质上分为不同类型——有些需要日复一日地坚持，有些只是定期提醒别忘了，有些排好队依次做完就行，还有些只是"有空再说"。把它们混在一起，反而让人焦虑。

## 四象限之外

大多数任务管理工具基于"紧急-重要"四象限，但在实际生活中，很多事情并不是靠紧急程度来区分的——每天背单词和每月交房租，哪个更紧急？它们根本不在同一个维度上。

我们从**任务的本质属性**出发，将日常事务分为四类：

---

## 一、习惯打卡

**定义**：需要每天重复做的事情，目标是养成习惯、保持节奏。

**核心特征**：
- 每天自动重置，不需要手动创建
- 做了就打勾，没做就是空的——简单直接
- 重点不是"完成任务"，而是"保持连续"
- 需要看到历史记录，感受积累的力量

**例子**：
- 背单词 / 多邻国
- 练字 / 画画
- 运动 / 喝水
- 写日记

**与传统 Todo 的区别**：传统做法是设一个"每日重复"任务，完成后生成下一个。问题是，如果昨天忘了点完成，今天就不会出现新的。而习惯打卡是**无状态**的——今天有没有打勾，只取决于今天，和昨天无关。

---

## 二、定期提醒

**定义**：固定周期触发的事情，不需要每天惦记，到了日子提醒一下就好。

**核心特征**：
- 不是任务，是**提醒**
- 不需要"完成"，只需要"知晓"
- 按固定周期出现：每月几号、每周几、每年几月几号
- 非激活期间完全不显示，不制造噪音

**例子**：
- 每月 19 号交房租
- 每周五倒垃圾
- 每年 3 月 15 号续保险

**与传统 Todo 的区别**：传统做法把提醒当任务，完成后计算下一次日期再创建。但提醒本质上不是"要做的事"，而是"别忘了的事"。它只需要在该出现的时候出现，点一下"已知晓"就消失。

---

## 三、任务流

**定义**：当前阶段需要按顺序完成的一组事情，做完一个接一个。

**核心特征**：
- 有明确的先后顺序
- 做完就消失，不留痕迹
- 可以随时往里加，可以拖拽排序
- 适合用来管理"今天要做的事"或"这周的计划"

**例子**：
- 1. 去超市买菜
- 2. 回家做饭
- 3. 学习 Godot 一小时
- 4. 写周报

**与传统 Todo 的区别**：传统 Todo 不区分顺序，所有未完成项堆在一起。任务流强调**序列感**——你不需要同时看到所有事情，只需要知道"下一个该做什么"。

---

## 四、待办池

**定义**：想做但不着急的事情，有空的时候从里面挑一个做。

**核心特征**：
- 没有截止日期，没有优先级
- 纯粹的"想法收集箱"
- 做了就标记完成，不想做了就删掉
- 不会产生任何压力

**例子**：
- 学 AI 绘画
- 整理书桌
- 试试新的咖啡店
- 看《三体》

**与传统 Todo 的区别**：传统做法把这些和其他任务混在一起，长期不做就成了"钉子户"，看着就有压力。待办池是一个独立的、低压力的空间——它就像一个"有空再看"的书签栏。

---

## 设计哲学

### 减法思维

传统任务管理工具追求功能全面：优先级、标签、子任务、截止日期、提醒时间……功能越多，使用成本越高，最终大多数人回到便签纸。

四类任务模型的核心是**用分类替代属性**。不需要给每个任务标优先级——习惯本身就是最高优先级（每天都要做），待办池本身就是最低优先级（有空再做）。不需要设截止日期——习惯按天重置，提醒按周期触发。

### 零配置自动化

- 习惯跨天自动重置，不需要手动操作
- 提醒到期自动出现，确认后自动消失
- 任务流完成即消失，不需要清理
- 无需设置时区——跟随系统本地时间

### 情绪友好

- 习惯用连续天数激励，而不是用"未完成"施压
- 提醒只在该出现时出现，不制造待办焦虑
- 待办池明确标记为"不紧急"，消除心理负担
- 任务流聚焦当下，不展示遥远的未来
