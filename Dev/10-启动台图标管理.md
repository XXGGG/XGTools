# 启动台图标管理

本章节记录启动台（Dock）的编辑功能和自定义图标管理功能的开发过程。

## 功能概述

在原有启动台 App 管理基础上，新增以下功能：

1. **编辑应用** — 每个 App 行增加编辑按钮（铅笔图标），可修改名称和图标
2. **图标管理** — 新增"图标管理"选项卡，支持上传自定义图标
3. **图片裁剪** — 上传图片后弹出 1:1 裁剪器，裁剪结果统一为 128x128 PNG
4. **SVG 支持** — SVG 文件直接渲染为 128x128 PNG 存储
5. **透明通道** — 保留 PNG/SVG 的 alpha 通道

## 技术方案

### Rust 后端

在 `dock_commands.rs` 中新增 4 个 Tauri Command：

| Command | 功能 |
|---------|------|
| `save_custom_icon` | 保存裁剪后的 PNG（base64）到 `icons/custom/` 目录 |
| `get_custom_icons` | 读取所有自定义图标，返回 `Vec<CustomIcon>` |
| `delete_custom_icon` | 删除指定图标文件及元数据 |
| `rename_custom_icon` | 修改图标的显示名称 |

**存储结构**：
```
AppData/
└── icons/
    └── custom/
        ├── meta.json          # 图标元数据 { icons: [{ id, name }] }
        ├── a1b2c3d4.png       # UUID 命名的图标文件
        └── e5f6g7h8.png
```

使用 `meta.json` 而不是文件名存储显示名称，方便重命名且支持特殊字符。

**依赖新增**：
- `uuid = { version = "1", features = ["v4"] }` — 生成唯一图标 ID

### 前端

#### 编辑对话框

点击铅笔按钮弹出编辑对话框，包含：
- 图标预览（当前图标）
- 名称输入框
- 路径显示（只读）
- "更换图标"按钮 → 从自定义图标库选择
- "恢复默认"按钮 → 重新从 exe 提取原始图标

#### 图标管理选项卡

新增第三个 Tab "图标管理"，功能：
- 上传按钮：支持 `.png`, `.jpg`, `.jpeg`, `.svg`, `.webp`
- 裁剪器：使用 `vue-advanced-cropper`，1:1 比例，裁剪后缩放到 128x128
- 图标网格：5 列展示，显示图标名称
- 双击名称可编辑（inline rename）
- 右上角删除按钮

#### 图片处理流程

```
用户选择文件
  ├─ SVG → readTextFile → btoa → Image → Canvas(128x128) → toDataURL → save
  └─ PNG/JPG/WebP → readFile → base64 → Cropper(1:1) → Canvas(128x128) → save
```

**依赖新增**：
- `vue-advanced-cropper` — Vue 3 图片裁剪组件

### 权限配置

`capabilities/default.json` 中新增：
```json
"fs:allow-read-file",
{ "identifier": "fs:scope", "allow": [{ "path": "**" }] }
```

需要通配符 `**` 才能读取用户通过 dialog 选择的任意路径文件。

## 开发过程中的问题与解决

### 1. 文件读取 API

Tauri v2 的 `@tauri-apps/plugin-fs` 提供了 `readFile`（二进制）和 `readTextFile`（文本）两个方法，不能用 `invoke('plugin:fs|read_file')` 这种低级调用方式。

### 2. 图标名称显示

最初图标用 UUID 命名文件，列表中直接显示 UUID（如 `a1b2c3d4`），用户体验差。改为使用 `meta.json` 存储 `{ id, name }` 映射，默认名称取上传文件的原始文件名（去后缀）。

### 3. 删除按钮被裁剪

图标卡片右上角的删除按钮用 `absolute -top-1 -right-1` 定位，被父容器 `overflow-hidden` 裁剪。解决方案：将按钮移到卡片内部 `top-0.5 right-0.5`，移除 `overflow-hidden`，外层 grid 加 `p-2`。

### 4. Dock 名称溢出

Dock overlay 中长名称（如"Visual Studio Code测试测试..."）超出图标宽度。解决方案：在 `AppIcon.vue` 的 name span 上加 `truncate` class 配合动态 `maxWidth` 样式绑定：
```vue
<span class="truncate" :style="{ maxWidth: (store.iconSize + 24) + 'px' }">
```

### 5. Tab 对齐

设置页的 Tab 按钮没有和下方内容区域对齐（内容区 `max-w-2xl mx-auto`）。在 Tab 切换器容器上加同样的 `max-w-2xl mx-auto w-full` 即可。
