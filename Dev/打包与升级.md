# 打包与升级

本章节介绍如何将 XGTools 打包为 Windows 安装程序，以及后续版本升级的流程。

## 打包构建

### 前置条件

- Node.js + pnpm
- Rust toolchain（`rustup` 安装）
- Windows 10/11（WebView2 Runtime 已内置）

### 构建命令

```bash
pnpm tauri build
```

这条命令会依次执行：

1. **前端构建** — 运行 `pnpm build`（Vite），输出到 `dist/`
2. **Rust 编译** — `cargo build --release`，编译后端
3. **打包安装程序** — 生成 Windows 安装包

### 构建产物

构建完成后，产物在 `src-tauri/target/release/bundle/` 下：

| 格式 | 路径 | 说明 |
|------|------|------|
| NSIS | `bundle/nsis/xgtools_x.y.z_x64-setup.exe` | 推荐，轻量安装器 |
| MSI | `bundle/msi/xgtools_x.y.z_x64_en-US.msi` | Windows Installer 格式 |
| 裸 exe | `target/release/xgtools.exe` | 不含安装器，需要系统有 WebView2 |

### 只构建 NSIS

如果只需要 NSIS 安装包（推荐）：

```bash
pnpm tauri build -b nsis
```

### NSIS 安装模式

在 `tauri.conf.json` 中配置：

```json
{
  "bundle": {
    "windows": {
      "nsis": {
        "installMode": "currentUser"
      }
    }
  }
}
```

| 模式 | 安装路径 | 需要管理员权限 |
|------|---------|--------------|
| `currentUser`（默认） | `%LOCALAPPDATA%\xgtools` | 否 |
| `perMachine` | `C:\Program Files\xgtools` | 是 |
| `both` | 用户安装时选择 | 是 |

推荐 `currentUser`，不需要管理员权限，用户体验更好。

## 版本号管理

版本号在两个地方定义，需要保持一致：

### 1. `src-tauri/tauri.conf.json`（主要）

```json
{
  "version": "0.1.0"
}
```

这是 Tauri 打包器读取的版本号，会写入安装包。

### 2. `src-tauri/Cargo.toml`（辅助）

```toml
[package]
version = "0.1.0"
```

如果 `tauri.conf.json` 没有 `version` 字段，Tauri 会回退读取这里。建议两边都改。

### 3. `package.json`（前端，可选）

```json
{
  "version": "0.1.0"
}
```

Tauri 不读取这个，但保持一致方便管理。

### 升级版本号的步骤

假设从 `0.1.0` 升级到 `0.2.0`：

1. 修改 `src-tauri/tauri.conf.json` 中的 `version` 为 `"0.2.0"`
2. 修改 `src-tauri/Cargo.toml` 中的 `version` 为 `"0.2.0"`
3. （可选）修改 `package.json` 中的 `version`
4. 运行 `pnpm tauri build`

## 升级安装（覆盖行为）

### 用户手动升级

用户拿到新版本的 `xgtools_0.2.0_x64-setup.exe`，直接运行安装即可。

**NSIS 安装器的行为：**

- **程序文件被覆盖** — exe、dll、资源文件全部替换为新版本
- **用户数据不受影响** — `%APPDATA%` 下的数据目录完全保留：
  - `apps.json`（应用列表）
  - `settings.json`（启动台设置）
  - `icons/`（提取的图标、自定义图标）
  - `store/`（tauri-plugin-store 的数据）
- **安装路径自动复用** — 检测到旧版本后，默认安装到同一目录
- **注册表自动更新** — 版本号、卸载信息等会更新

**简单来说：直接装新版就行，不需要先卸载，用户数据不会丢。**

### 什么时候数据会被清除？

只有用户 **卸载** 应用时，如果勾选了"删除应用数据"选项，`%APPDATA%` 下的数据才会被删除。正常覆盖安装不会触发。

## 自动更新（可选）

Tauri v2 提供了内置的 updater 插件，可以实现应用内自动检查和安装更新。如果将来需要，步骤如下：

### 1. 生成签名密钥

```bash
pnpm tauri signer generate -w ~/.tauri/xgtools.key
```

生成公钥（写入配置）和私钥（构建时用，妥善保管）。

### 2. 配置 `tauri.conf.json`

```json
{
  "bundle": {
    "createUpdaterArtifacts": true
  },
  "plugins": {
    "updater": {
      "pubkey": "<公钥内容>",
      "endpoints": [
        "https://your-server.com/update/latest.json"
      ]
    }
  }
}
```

### 3. 构建时提供私钥

```bash
TAURI_SIGNING_PRIVATE_KEY=<私钥内容> pnpm tauri build
```

构建会额外生成 `.sig` 签名文件。

### 4. 托管更新 JSON

在服务器上放一个 JSON 文件，内容：

```json
{
  "version": "0.2.0",
  "notes": "新功能：xxx",
  "pub_date": "2026-02-21T00:00:00Z",
  "platforms": {
    "windows-x86_64": {
      "url": "https://your-server.com/releases/xgtools_0.2.0_x64-setup.exe",
      "signature": "<.sig 文件内容>"
    }
  }
}
```

没有新版本时返回 HTTP 204。

### 5. 前端代码

```typescript
import { check } from '@tauri-apps/plugin-updater'
import { relaunch } from '@tauri-apps/plugin-process'

const update = await check()
if (update) {
  await update.downloadAndInstall()
  await relaunch()
}
```

> **注意：** 如果丢失了私钥，就无法再向已安装用户推送更新。一定要备份好密钥。

## 常用命令速查

```bash
# 开发模式
pnpm tauri dev

# 构建 NSIS 安装包
pnpm tauri build -b nsis

# 构建全部格式
pnpm tauri build

# 生成更新签名密钥
pnpm tauri signer generate -w ~/.tauri/xgtools.key
```

## 分发清单

发布新版本时，需要提供给用户的文件：

- `xgtools_x.y.z_x64-setup.exe` — NSIS 安装程序（推荐）
- 如果启用了自动更新，还需要上传 `.sig` 签名文件和更新 JSON
